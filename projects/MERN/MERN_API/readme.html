Readme

API structure

  • app.js requires config.js and initializes app

  • Models create schemas for the collections

  • routers call cotrollers depending on the route

  • controllers take the request and find/create/update/delete collections using models

  • Some request functions use middlewares for user verification / image upload

Users

Username is unique, so it cannot be changed (you can change it in the user model)

email, bio and password can be changed. If value not sent in request, it will update the rest (update email or bio but not password...)

Posts

Posts have: required: (title,user (owner id) ),optional: (description, image) description,

Images

now it is specific for posts, but can be adapted easily (I think)

By default, images are saved in public/uploads/<username>/posts/

To see an image, send get request to "/images/<username>/<post id>"
